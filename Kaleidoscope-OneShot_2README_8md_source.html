12 One-shots are a new kind of behaviour for your standard modifier and momentary
13 layer keys: instead of having to hold them while pressing other keys, they can
14 be tapped and released, and will remain active until any other key is pressed.
15 In short, they turn `Shift, A` into `Shift+A`, and `Fn, 1` to `Fn+1`. The main
16 advantage is that this allows us to place the modifiers and layer keys to
17 positions that would otherwise be awkward when chording. Nevertheless, they
18 still act as normal when held, that behaviour is not lost.
19 
20 Furthermore, if a one-shot key is tapped two times in quick succession, it
21 becomes sticky, and remains active until disabled with a third tap. This can be
22 useful when one needs to input a number of keys with the modifier or layer
23 active, and still does not wish to hold the key down. If this feature is
24 undesirable, unset the `OneShot.double_tap_sticky` property (see later).
25 
26 To make multi-modifier, or multi-layer shortcuts possible, one-shot keys remain
27 active if another one-shot of the same type is tapped, so `Ctrl, Alt, b` becomes
28 `Ctrl+Alt+b`, and `L1, L2, c` is turned into `L1+L2+c`. Furthermore, modifiers
29 and other layer keys do not cancel the one-shot effect, either.

31 ## Using the plugin
32 
33 After adding one-shot keys to the keymap, all one needs to do, is enable the
34 plugin:
35 
36 ```c++
37 #include <Kaleidoscope.h>
38 #include <Kaleidoscope-OneShot.h>
39 #include <kaleidoscope/hid.h>
40 
41 // somewhere in the keymap...
42 OSM(LeftControl), OSL(_FN)
43 
44 void setup() {
45  Kaleidoscope.use(&OneShot);
46 
47  Kaleidoscope.setup();
48 }
49 ```

51 ## Keymap markup
52 
53 There are two macros the plugin provides:
54 
55 ### `OSM(mod)`
56 
57 > A macro that takes a single argument, the name of the modifier: `LeftControl`,
58 > `LeftShift`, `LeftAlt`, `LeftGui` or their right-side variant. When marked up
59 > with this macro, the modifier will act as a one-shot modifier.
60 
61 ### `OSL(layer)`
62 
63 > Takes a layer number as argument, and sets up the key to act as a one-shot
64 > layer key.
65 >
66 > Please note that while `Kaleidoscope` supports more, one-shot layers are
67 > limited to 8 layers only.

69 ## Plugin methods
70 
71 The plugin provides one object, `OneShot`, which implements both one-shot
72 modifiers and one-shot layer keys. It has the following methods:
73 
74 ### `.isActive()`
75 
76 > Returns if any one-shot key is in flight. This makes it possible to
77 > differentiate between having a modifier or layer active, versus having them
78 > active only until after the next key getting pressed. And this, in turn, is
79 > useful for macros that need to fiddle with either modifier or layer state: if
80 > one-shots are not active, they need not restore the original state.
81 
82 ### `.isSticky(key)`
83 
84 > Returns if the key is currently sticky.
85 
86 ### `.isModifierActive(key)`
87 
88 > Returns if the modifier `key` has a one-shot state active. Use this together
89 > with `hid::isModifierKeyActive` to catch cases where a one-shot modifier is
90 > active, but not registered yet.
91 
92 ### `.cancel([with_stickies])`
93 
94 > The `cancel()` method can be used to cancel any pending one-shot effects,
95 > useful when one changed their minds, and does not wish to wait for the
96 > timeout.
97 >
98 > The optional `with_stickies` argument, if set to `true`, will also cancel
99 > sticky one-shot effects. If omitted, it defaults to `false`, and not canceling
100 > stickies.
101 
102 ## Plugin properties
103 
104 Along with the methods listed above, the `OneShot` object has the following
105 properties too:
106 
107 ### `.time_out`
108 
109 > Set this property to the number of milliseconds to wait before timing out and
110 > cancelling the one-shot effect (unless interrupted or cancelled before by any
111 > other means).
112 >
113 > Defaults to 2500.
114 
115 ### `.hold_time_out`
116 
117 > Set this property to the number of milliseconds to wait before considering a
118 > held one-shot key as intentionally held. In this case, the one-shot effect
119 > will not trigger when the key is released. In other words, holding a one-shot
120 > key at least this long, and then releasing it, will not trigger the one-shot
121 > effect.
122 >
123 > Defaults to 200.
124 
125 ### `.double_tap_sticky`
126 
127 > Set this boolean property to make the plugin treat a double-tap of a one-shot
128 > key as making it sticky until a third tap. Setting it to `false` disables this
129 > behaviour, in which case double-tapping a one-shot modifier will just restart
130 > the timer.
131 >
132 > Defaults to `true`.
